About

The Science of Remembering

Named for Mnemosyne — the Greek Titaness of memory and mother of the Muses — our institute investigates how language models represent knowledge geometrically, how to generate training data that is formally verified rather than merely plausible, and how to build engineering tools that embody these principles in practice.

🔆

Geometric AI Theory

Typed embeddings as geometric priors. We study how frozen type structure introduces constraint into model outputs — making certain failure modes geometrically impossible, not just statistically unlikely.

✅

Formal Verification

Generating training data that is formally verified by SymPy and Lean before it enters a model. The ChatDB proof engine targets the IMO corpus — 3–4 million typed, verified reward signals.

♞

Empirical Benchmarking

The Oracle Trust Calibration Framework uses chess as a measurement instrument — pitting frontier LLMs against Stockfish to observe, categorize, and publish cognitive failure modes.

🛠

Engineering Tools

ProveCalc brings the constraint-as-information thesis into practice: verified engineering calculations with dimensional analysis, PE-stamp-ready audit trails, and perpetual desktop licensing.

Research

Active Programs

Our work spans AI architecture theory, formal verification, empirical benchmarking, and engineering tools — unified by a single thesis: hallucination is a constraint problem.

Active

ANLU — Architecturally Native Language Understanding

Typed embeddings as geometric priors. Frozen type embeddings introduce geometric impossibility — not just statistical unlikelihood — into language model outputs. Track A (A0–A3) complete. Track B in progress: Hierarchy of Controls, Constraint as Information, Alignment as Topology.

LLM Architecture Geometric Priors Type Theory
Active

ChatDB — Verified Proof Engine

Autonomous proof engine targeting the IMO corpus. LLMs propose steps. SymPy and Lean verify. An obligation DAG tracks what remains to be proven. Key finding: 19/19 locally verified steps on IMO 2025 P3 still produced the wrong answer — verification is not exploration.

Formal Verification Training Data Rust + Python
Active

OTCF — Oracle Trust Calibration Framework

Frontier LLMs vs. Stockfish at controlled ELO ratings. Five cognitive failure modes documented: forced-move blindness, endgame collapse, the Gemini paradox, Nemotron pattern blindness, and context saturation degradation. Chess as a proxy for general reasoning failure.

LLM Benchmarking Chess Cognitive Failure Modes
Active

ProveCalc — Constraint-Based Engineering Calculations

Engineering calculation platform where the LLM proposes and the engine (SymPy + Pint) validates. Four-gate verification: unit consistency, constraint satisfaction, numeric residual, sanity checks. PE-stamp-ready audit trail. Practical application of the constraint-as-information thesis.
